# Occupation Pages

## Core Principle

Pages MUST be tool-oriented and genuinely useful, not article-oriented. Three things make each page valuable:
1. **Real data** — task-by-task breakdown, salary vs national median, BLS projections
2. **Specific AI tools** — named, linked tools actively automating this role (not generic AI mentions)
3. **Embedded risk calculator** — the HeroRiskGame widget, pre-mapped to the job's category

Never add fluff content. Every section must answer a real question the visitor has.

## Page Sections (exact order)

### 1. Breadcrumb nav
```tsx
<nav aria-label="Breadcrumb">
  Home → Browse Jobs → {occ.title}
</nav>
```
Schema: `BreadcrumbList` (see Schema Markup section).

---

### 2. Hero
- Eyebrow: `AI Risk Assessment · {year}`
- H1: `Will AI Replace\n{occ.title}s?`
- **Verdict paragraph** — 3-sentence GEO-optimised direct answer generated by `verdictText()`. Must appear immediately after H1, before any stats. This is what AI search engines extract and cite.

---

### 3. Stats bar (3-column)
| Automation Risk | Median Salary | 10-Year Outlook |
|---|---|---|
| `{score}%` + risk level sub-label | `$XK` + above/below US median | `+X%` / `-X%` + trend label |

US national median reference: **$59,228** (BLS 2023–24). Always show the delta.

---

### 4. Risk badge
`<Badge label="{risk_level} Automation Risk" risk={occ.risk_level} />`

---

### 5. Task-by-task breakdown
- Section intro text explaining what Very High / High / Low risk means
- Progress bars per task (red = Very High/High, amber = Moderate, teal = Low/Minimal)
- **At risk now / Your protection callout** — two-column summary boxes showing top 3 high-risk tasks (red ×) and top 3 low-risk tasks (green ✓). Only render if both groups have entries.

---

### 6. AI Tools section ("What's changing")
Heading: `AI Tools Already Automating {occ.title} Work`

Rules:
- Always 5 named, real tools per occupation — no generic placeholders
- Each tool has: numbered badge, **name**, description of what it specifically does in this role, and a **"Visit →" link** to the tool's official website
- Link format: `<a href={tool.url} target="_blank" rel="noopener noreferrer">Visit →</a>`
- The `url` field in `AI_TOOLS_BY_SLUG` is the official website. **Future affiliate links go here** — only the `url` value needs updating, not the template.

Data structure (all three fields required):
```typescript
const AI_TOOLS_BY_SLUG: Record<string, { name: string; what: string; url: string }[]> = {
  paralegal: [
    { name: 'Harvey AI', what: '...', url: 'https://www.harvey.ai' },
    // ... 4 more
  ],
  // ... all 10 occupations
};
```

**When adding a new occupation**, add a matching entry to `AI_TOOLS_BY_SLUG` with 5 tools and their official URLs. The fallback (for slugs not in the map) is acceptable for new occupations until they get proper entries.

### 7. Risk calculator (HeroRiskGame — IntroPhase)
Uses `<HeroRiskGame prefilledJobId={titleToJobId(occ.title)} />`.

The `titleToJobId` function maps occupation titles to HeroRiskGame job category IDs. When `prefilledJobId` is set, the calculator shows:
- A single primary button "Check my risk score" (full-width, `bg-fire`, centered text, no icon)
- "Or" separator
- "Upload your resume to calculate your score" button

Clicking the button pre-selects the job category and starts at Q1 (skipping the pick grid). See `saveyojob_generator-ui.md` for full HeroRiskGame documentation.

---

### 8. Skills that protect you
Two-column grid:
- **Left:** Core domain skills from `occ.core_skills`, with importance % and progress bar
- **Right:** 4 human skills AI can't replicate (static list, same across all pages): strategic judgment, stakeholder trust, cross-domain synthesis, ethical decision-making
- Affiliate courses from `occ.affiliate_courses` if present (link opens in new tab)

---

### 9. Career protection steps (5 steps)
Generated by `careerSteps(occ)`. Content is derived from the occupation's low-risk tasks and top core skills — not hardcoded. Steps are always in this order:
1. Own the AI-resistant tasks (references actual low-risk tasks from data)
2. Use the tools disrupting the field (references top 2 tools from `getAiTools`)
3. Deepen highest-leverage skills (references top core skills)
4. Signal AI fluency to the job market
5. Build a 90-day reskilling plan

---

### 10. Job outlook
- Salary card: median wage, vs national median ($59,228 delta)
- Employment card: workforce size, BLS 10-year projection %, SOC code
- Contextual paragraph — changes based on growth/decline: <-5% gets decline language, >10% gets growth language, else stable

---

### 12. FAQ (6 questions)
Generated by `buildFaqs(occ, score)`. Questions are always these six — answers use real data:
1. Will AI replace {title}s?
2. Which {title} tasks will AI automate first?
3. What skills do {title}s need in {year} and beyond?
4. How much do {title}s earn and is the salary outlook positive?
5. Is a {title} career still worth pursuing in {year}?
6. What should a {title} do in the next 6 months to stay ahead?

Each answer references actual data fields (tasks, wages, growth, tool names). Generic answers are not acceptable.

---

### 13. Data sources note
Always present at the bottom, before the footer:
> Automation probability scores are derived from Frey & Osborne (2013), *The Future of Employment*, University of Oxford. Employment counts, median wages and 10-year projections from the U.S. Bureau of Labor Statistics (BBS) Occupational Outlook Handbook, 2023–24. Broader automation impact from McKinsey Global Institute, *Jobs Lost, Jobs Gained* (2017). Risk assessments reflect probabilities of task-level automation, not whole-job elimination.

## Content Generators

All content is derived from occupation data — no hardcoded occupation-specific strings outside the data maps.

### `verdictText(occ, score)`
3-sentence direct answer tailored to risk level. Rules:
- Very High / High: urgent tone, names specific tools, references employment trend
- Moderate: "inflection point" framing, acknowledges both risk and resilience
- Low: reassuring but not complacent, still mentions AI adoption

### `careerSteps(occ)`
Reads `occ.tasks` filtered to Low/Minimal risk, `occ.core_skills` sorted by importance desc, and `getAiTools(occ.slug)[0–1]`. Returns 5 steps with step title + detail paragraph.

### `buildFaqs(occ, score)`
Uses `occ.tasks`, `occ.core_skills`, `occ.median_annual_wage`, `occ.ten_year_growth_pct`, and `getAiTools`. The national median constant is `59228`. Always computes wage delta against this.

## Schema Markup

All three schemas are rendered as a single `@graph` in one `<script type="application/ld+json">` at the top of `<main>`:

```typescript
const schema = {
  '@context': 'https://schema.org',
  '@graph': [
    {
      '@type': 'BreadcrumbList',
      itemListElement: [
        { '@type': 'ListItem', position: 1, name: 'Home',        item: `${siteUrl}/` },
        { '@type': 'ListItem', position: 2, name: 'Browse Jobs', item: `${siteUrl}/jobs/` },
        { '@type': 'ListItem', position: 3, name: `${occ.title} AI Risk`, item: `${siteUrl}/jobs/${occ.slug}/` },
      ],
    },
    {
      '@type': 'FAQPage',
      mainEntity: faqs.map(f => ({
        '@type': 'Question',
        name: f.q,
        acceptedAnswer: { '@type': 'Answer', text: f.a },
      })),
    },
    {
      '@type': 'WebPage',
      name: `Will AI Replace ${occ.title}s? ${score}% Automation Risk Score`,
      about: {
        '@type': 'Occupation',
        name: occ.title,
        occupationalCategory: occ.soc_code,
        estimatedSalary: {
          '@type': 'MonetaryAmountDistribution',
          name: 'median annual wage',
          currency: 'USD',
          duration: 'P1Y',
          median: occ.median_annual_wage,
        },
      },
    },
  ],
};
```

## Risk Colour Helpers

function riskColor(level: string | null) {
  // Returns Tailwind text class
  'Very High' → 'text-critical'   // #DC2626
  'High'      → 'text-warn'       // #B45309
  'Moderate'  → 'text-amber-400'
  'Low'       → 'text-fire'       // #0C526D
}

function riskBorderColor(level: string | null): string {
  // Returns hex for inline styles (numbered badges, step circles)

  'High'      → '#B45309'
  'Moderate'  → '#D97706'
  'Low'       → '#0C526D'
  default     → '#6B7280'
}

## Internal Linking Rules

- Every slug page: breadcrumb links Home → Browse Jobs → current
- Every slug page: related occupations section (3 cards)
- Footer: "All Jobs" → /jobs/, "About", "Privacy"
- DO NOT add external links except in the AI Tools section (Visit → links) and affiliate course links

## Adding a New Occupation

Checklist:
1. Add entry to `SEED_OCCUPATIONS` in `data/seed-data.ts` with all required fields
2. Add 5 tools to `AI_TOOLS_BY_SLUG` with `name`, `what`, and `url` (official website)
3. Add slug to `RELATED_SLUGS` bidirectionally
4. Add slug to `titleToJobId()` regex if needed
5. Run `npx tsc --noEmit` — zero errors required before shipping
